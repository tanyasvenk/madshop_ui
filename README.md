<img width="330" height="672" alt="image" src="https://github.com/user-attachments/assets/c225dab3-25da-49dd-b585-950fc73116cd" /># MadShop UI (Flutter)

Этот проект представляет собой мобильное приложение, реализующее пользовательский интерфейс онлайн-магазина. Разработка велась с акцентом на вёрстку экранов, навигацию и базовые принципы UI/UX.

---

## Описание приложения

### Регистрация (Sign Up)
Экран содержит поля для ввода телефона, email и пароля. Предназначен для первичного входа пользователя в систему и выполнен в единой стилистике проекта.

### Вход (Login)
Позволяет авторизоваться с помощью email. Структура и оформление экрана соответствуют общему дизайну приложения.

### Пароль (Password)
Дополнительное окно, которое следует после логина. Предназначено для ввода пароля.

### Главный экран магазина (Shop)
Основная часть приложения. На экране размещены:
- сетка товаров с изображениями, названием и ценой;
- строка поиска;
- нижняя панель навигации;
- карточки товаров, выполненные в модульном формате.

### Экран избранного (Favorites)
Отображает товары, отмеченные пользователем. Сохраняет структуру карточек и общую сетку из экрана Shop.

### Экран корзины (Cart)
Содержит список добавленных товаров. Каждая карточка включает изображение, цену, название, управление количеством и иконку удаления.  
Внизу расположена закреплённая панель с общей суммой и кнопкой оформления заказа.

---

## Навигация

В приложении используется `BottomNavigationBar`, позволяющий переключаться между Shop, Favorites и Cart.  
Активный пункт выделяется цветом и подчёркивающей линией под иконкой.

---

## Используемые элементы

В проекте применены следующие компоненты Flutter:

- `StatelessWidget`, `StatefulWidget`
- `GridView`, `Stack`, `Positioned`
- `ClipRRect`, `BoxDecoration`, `Container`
- `BottomNavigationBar`
- `TextField` (поиск)
- `SvgPicture` (отображение SVG-иконок)
- Собственные темы цветов и текста (`theme/colors.dart`, `theme/text_styles.dart`)
- `Image.asset` для изображений товаров

---

## Отличия от макета

Некоторые элементы имеют небольшие отличия от изначального дизайна. Это связано с использованием стандартных Flutter-компонентов, различиями в размерах экранов и некоторыми адаптациями для стабильной работы верстки.  
Визуально приложение сохраняет общий стиль и логику макета.
